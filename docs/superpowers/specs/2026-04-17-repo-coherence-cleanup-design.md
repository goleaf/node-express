# Repo Coherence Cleanup Design

## Goal

Make the repository match the current application reality without adding new product features. The app is SQLite-backed, uses build outputs generated by Vite, and should not keep local runtime artifacts or legacy JSON-store references under source control.

## Scope

- Update repository documentation so it describes the implemented app instead of an older JSON-backed version.
- Treat SQLite as the only persisted application store.
- Stop tracking local runtime and generated files that should be recreated locally:
  - SQLite database files and WAL/SHM companions
  - Vite production build output under `public/dist/`
  - Playwright capture artifacts
  - Temporary request-log text files created during manual verification
- Remove legacy JSON-store residue such as `data/todos.json`.

## Non-Goals

- No new UI or API features.
- No attempt to make the old README promises true.
- No behavioral refactor of the task, auth, onboarding, or dashboard flows beyond what is required for coherence cleanup.

## Design Decisions

### Persistence

`src/config/database.js` remains the runtime source of truth. Fresh checkouts should continue bootstrapping schema and migrations automatically when the app starts, so the repository does not need checked-in SQLite databases.

### Build Artifacts

`public/dist/` is a generated output directory. Development should continue to work with `npm run dev`, and production-style runs should require `npm run build` before `npm start`.

### Documentation

`README.md` should only describe verified routes and capabilities:

- Authentication and password-reset flows
- Onboarding
- Dashboard
- Tasks, subtasks, categories, tags, search, notifications, and profile/preferences
- Current scripts and storage behavior

It should stop advertising routes or features that do not exist in the codebase.

## Verification

- `npm test`
- `npm run build`

## Risks

- Removing tracked runtime/build files will change the working tree significantly, but this is intentional cleanup.
- Local runs after cleanup will recreate ignored SQLite/build artifacts; that is expected and should no longer pollute git status.
